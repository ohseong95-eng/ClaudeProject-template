# 신규 프로젝트 생성 가이드

> CLAUDE.md에서 분리된 참조 문서. 프로젝트 생성 시 참조한다.

---

## 1. 에이전트 설계 방식 결정 — 반드시 사용자에게 질문할 것

프로젝트 생성 전 **항상** 아래 질문을 사용자에게 한다.

> "이 프로젝트의 에이전트를 **capability 기반**으로 설계할까요?
> capability 기반은 fix_action + layer 조합으로 에이전트를 정의하여
> 도메인에 무관하게 재사용하는 방식입니다.
> 대안으로 도메인/기능별 전담 에이전트 방식도 가능합니다."

| 방식 | 특징 | 적합한 경우 |
|------|------|------------|
| **capability 기반** (기본) | fix_action + layer 조합, 재사용성 높음 | 버그 수정·검증처럼 반복 패턴이 많은 작업 |
| **도메인/기능 전담** | 에이전트가 특정 역할에 집중 | 파이프라인 단계가 명확히 분리되는 작업 |

사용자 답변을 받은 후 해당 방식에 맞게 에이전트를 설계한다.
**사용자 답변 없이 에이전트 설계를 진행하지 않는다.**

---

## 2. 생성 체크리스트

신규 프로젝트 생성 시 아래 항목을 순서대로 완료한다.

```
[ ] 1. 에이전트 설계 방식 사용자에게 질문하고 답변 수신
[ ] 2. 프로젝트 디렉토리 및 필수 하위 폴더 생성
[ ] 2-1. make/ 디렉토리 생성 및 설계 문서 저장
[ ] 3. CLAUDE.md 작성 (통일 템플릿 — 아래 참조)
[ ] 3-1. PROFILE.md 작성 (프로젝트별 세부사항 — 아래 템플릿 참조)
[ ] 3-2. .claude/config.md 생성 (docs/public/config-template.md 참조)
[ ] 4. 에이전트 설계 방식에 맞게 .claude/agents/ 구성
[ ] 4-1. 프로젝트에 필요한 skill 디렉토리를 .claude/skills/{skill-name}/SKILL.md 형식으로 생성 (docs/public/skill-guide.md 참조)
[ ] 5. `docs/project-registry.md` 프로젝트 레지스트리에 등록
```

> **참고**: `save-session`, `finalize-report` 등 공통 커맨드는 루트에서 상속된다. 프로젝트 전용 커맨드가 필요한 경우에만 `.claude/commands/`에 추가한다.

## 3. 필수 디렉토리 구조

모든 프로젝트는 아래 디렉토리를 반드시 포함한다.

```
{ProjectName}/
├── CLAUDE.md            ← 통일 템플릿 (모든 프로젝트 동일)
├── PROFILE.md           ← 프로젝트별 세부사항 (워크플로우, 특이사항)
├── .claude/
│   ├── config.md        ← 프로젝트 설정 (max_retry_count 등)
│   ├── agents/          ← 에이전트 파일 (.md)
│   ├── skills/          ← 에이전트 공통 기능 모듈 (디렉토리 기반)
│   │   └── {skill-name}/SKILL.md  ← 각 skill은 디렉토리 + SKILL.md
│   └── commands/        ← 슬래시 커맨드 (save-session.md, finalize-report.md 필수)
├── input/               ← 다른 프로젝트로부터 받은 입력 데이터
├── make/                ← 설계 문서 + 기능 계획 + 변경 이력
├── handoffs/            ← 완료된 HANDOFF.md 아카이브
├── output/              ← 사람이 읽는 최종 보고서
├── pending/             ← 삭제 대기·변경 제안 (운영용)
├── sessions/            ← 대화 세션 기록
├── todo/                ← 미해결 작업 항목
└── transactions/        ← sub-agent 간 데이터 공유 (SESSION_ID 기반 파일)
```

프로젝트 성격에 따라 추가 디렉토리를 자유롭게 둘 수 있다.

## 4. CLAUDE.md 통일 템플릿

모든 서브 프로젝트의 CLAUDE.md는 아래와 **동일하게** 작성한다. 프로젝트명만 변경한다.

```markdown
# CLAUDE.md — {프로젝트명}

> 프로젝트별 세부사항은 `PROFILE.md`를 참조한다.

## 필수 참조

작업 시작 전 반드시 아래 파일을 읽는다:

| 파일 | 용도 |
|------|------|
| `PROFILE.md` | 프로젝트 워크플로우, 특이사항, 제약사항 |
| `.claude/config.md` | 프로젝트 설정값 (경로, 재시도 횟수 등) |

## Permission & Safety

- 수정 대상 경로는 `.claude/config.md`에서 참조한다
- 모든 분석·수정 결과는 반드시 파일로 저장한다 (대화창 출력만은 금지)
- 저장 후 사용자에게 파일 경로를 알린다
```

---

## 5. PROFILE.md 템플릿

프로젝트별 세부사항을 기술한다. CLAUDE.md가 이 파일을 반드시 참조한다.

```markdown
# PROFILE — {프로젝트명}

## 프로젝트 설명
(이 프로젝트가 무엇을 하는지 1~3문장)

## 입력 소스
(config.md에서 참조하는 입력 경로. 없으면 "없음")

## Workflow
(이 프로젝트의 작업 절차를 단계별로 기술)

## 특이사항
(이 프로젝트만의 제약사항, 주의점, 특수 도구 사용법 등)
```
