# 프로젝트 목록 조회

`docs/project-registry.md`를 읽어 화면에 출력한다.

## 출력 규칙

- 도구 호출 과정, 중간 설명, 부가 대화를 일절 출력하지 않는다
- 오직 아래 "출력 형식"의 결과 표만 출력한다

## 실행 절차

1. `docs/project-registry.md`를 읽는다
2. 각 프로젝트의 디렉토리 존재 여부, `.claude/config.md` 유무를 확인한다
3. 아래 형식으로 화면에 출력한다

## 출력 형식

```
# 프로젝트 레지스트리

| # | 프로젝트 | 역할 | 상태 | config |
|---|---------|------|------|--------|
| 1 | `ClaudeProject_example/` | 예시 프로젝트 | [ON] 활성 | O |

---

총 N개 프로젝트 (활성 N / 사용 금지 N)
```

## 상태 표기

- `[ON]` — 활성 프로젝트
- `[--]` — 사용 금지 프로젝트

## config 표기

- `O` — 파일 존재
- `X` — 파일 없음
