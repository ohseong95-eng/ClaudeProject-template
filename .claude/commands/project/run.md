# 서브 프로젝트 실행

서브 프로젝트를 선택하고, 에이전트를 지정하여 **현재 대화 컨텍스트에서 직접** 실행한다.
Sub-agent(Task 도구)를 사용하지 않고, 에이전트 프롬프트의 절차를 직접 수행한다.

## 인자

$ARGUMENTS — 프로젝트 이름 또는 번호 (선택). 생략하면 목록에서 선택.

## 출력 규칙

- 중간 설명 없이 진행 상황과 결과만 출력한다

## 실행 절차

### Step 1 — 프로젝트 선택

1. `docs/project-registry.md`를 읽어 활성(`[ON]`) 프로젝트 목록을 보여준다
2. `$ARGUMENTS`가 있으면 해당 프로젝트를 자동 선택한다
3. 없으면 사용자에게 프로젝트를 선택하게 한다
4. 선택한 프로젝트가 `[--] 사용 금지`이면 경고 후 중단한다

### Step 2 — 에이전트 선택

1. `{프로젝트}/.claude/agents/*.agent-v1.0.md` 파일을 Glob으로 수집한다
2. 각 에이전트의 YAML frontmatter에서 `name`, `description`을 읽는다
3. 번호 목록으로 보여준다:
   ```
   # {프로젝트} 에이전트 목록
   | # | 에이전트 | 설명 |
   |---|---------|------|
   | 1 | scan-discover-endpoints | 엔드포인트 스캔 |
   | 2 | extract-analyze-auth | 인증 분석 |
   ```
4. 사용자에게 실행할 에이전트를 선택하게 한다

### Step 3 — 작업 내용 입력

사용자에게 질문한다:
- **작업 목표**: 이 에이전트에게 무엇을 시킬 것인지
- **추가 컨텍스트**: 필요한 배경 정보 (선택)

### Step 4 — SESSION_ID 발급

1. `{프로젝트}/transactions/` 폴더에서 오늘 날짜(`YYYYMMDD`) 파일 수를 센다
2. `SESSION_ID = YYYYMMDD-{4자리 순번}` (예: `20260223-0001`)

### Step 5 — 에이전트 프롬프트 로드 및 직접 실행

1. 선택한 에이전트 파일(`{프로젝트}/.claude/agents/{에이전트}.agent-v1.0.md`)을 읽는다
2. 에이전트의 `## Skills Used` 섹션이 있으면 참조된 skill 파일들을 함께 읽는다
3. 프로젝트의 `.claude/config.md`를 읽어 설정값을 파악한다
4. 에이전트 프롬프트의 절차(Procedure)를 **현재 대화에서 직접 수행**한다
   - 에이전트가 요구하는 입력(SESSION_ID, 파일 경로 등)을 직접 전달한다
   - 각 단계의 결과를 `{프로젝트}/transactions/` 또는 `{프로젝트}/output/`에 저장한다

### Step 6 — 결과 보고

실행 완료 후 결과를 요약한다:

```
## 실행 결과

| 항목 | 값 |
|------|-----|
| 프로젝트 | {프로젝트} |
| 에이전트 | {에이전트 이름} |
| SESSION_ID | {SESSION_ID} |
| 출력 파일 | {생성된 파일 경로} |
| 상태 | 성공 / 실패 |
```
