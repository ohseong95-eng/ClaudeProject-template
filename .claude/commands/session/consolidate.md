# 세션 데이터 통합 정리

세션 파일을 분석하고, 현재 프로젝트 상태와 비교하여 불필요한 데이터를 제거한 뒤, 하나의 안정적인(stable) 세션 파일로 통합한다.

**`session:merge`와의 차이:** merge는 단순 병합. 이 커맨드는 프로젝트 현재 상태를 기준으로 유효성을 판단하여 정리한다.

## 실행 절차

### Phase 1 — 스캔 및 요약

1. `sessions/*.md` 파일을 날짜순으로 정렬한다
2. 각 세션 파일을 읽고 아래를 추출한다:
   - 주요 주제 (1줄)
   - 결정된 사항
   - 완료된 작업
   - 미해결 사항
   - 참조 파일 목록
3. 요약 테이블을 사용자에게 보여준다:

```
## Phase 1: 세션 스캔 결과

| # | 파일 | 줄수 | 주제 | 상태 |
|---|------|------|------|------|
| 1 | 2026-02-20-xxx.md | 1868 | 프로젝트 초기 설계 | 판정 대기 |
| 2 | 2026-02-22-xxx.md | 5259 | 에이전트 명명 규칙 정비 | 판정 대기 |
...
```

### Phase 2 — 현재 프로젝트와 비교

각 세션의 내용을 현재 프로젝트 상태와 비교하여 판정한다:

**판정 기준:**

| 판정 | 의미 | 조건 |
|------|------|------|
| `obsolete` | 불필요 | 세션의 결정/작업이 이후 세션에서 덮어쓰거나 삭제됨 |
| `superseded` | 대체됨 | 같은 주제를 다른 세션에서 더 최신으로 다룸 |
| `partial` | 일부 유효 | 일부 결정/미해결 사항만 아직 유효 |
| `active` | 유효 | 미해결 사항이 아직 남아있거나 최신 세션 |

**비교 항목:**
- 세션에서 수정한 파일이 현재도 존재하는가?
- 세션의 결정 사항이 현재 CLAUDE.md/config에 반영되었는가?
- 세션의 미해결 사항이 이후 세션에서 해결되었는가?
- 세션에서 생성한 에이전트/스킬이 현재 존재하는가?

판정 결과를 사용자에게 보여준다:

```
## Phase 2: 판정 결과

| # | 파일 | 판정 | 사유 |
|---|------|------|------|
| 1 | 2026-02-20-xxx.md | obsolete | 모든 결정이 이후 세션에서 덮어씀 |
| 2 | 2026-02-22-xxx.md | superseded | 같은 주제를 #5에서 재작업 |
| 3 | 2026-02-23-xxx.md | active | 미해결 사항 1건 잔존 |

삭제 대상: #1, #2 (obsolete + superseded)
통합 대상: #3 (active + partial)

진행하시겠습니까?
```

### Phase 3 — 통합 및 정리

사용자 승인 후:

1. `active` + `partial` 세션에서 **아직 유효한 내용만** 추출한다:
   - 현재 유효한 결정 사항
   - 아직 남은 미해결 사항
   - 현재 존재하는 파일에 대한 참조만 유지
   - 이미 완료된 작업은 1줄 요약으로 압축

2. `sessions/YYYY-MM-DD-stable.md`로 저장한다:

```markdown
# Stable 세션 데이터: YYYY-MM-DD

> 통합일: YYYY-MM-DD
> 원본: N개 세션 → 1개 통합
> 기간: YYYY-MM-DD ~ YYYY-MM-DD

## 현재 유효한 결정 사항
- (현재 프로젝트에 반영된 주요 결정만)

## 완료 작업 요약
- (압축된 1줄 요약 목록)

## 미해결 사항
- (아직 처리되지 않은 항목)

## 핵심 참조
- (현재 존재하는 주요 파일만)
```

3. `obsolete` + `superseded` 세션 파일을 `pending/pending-delete.md`에 등록한다
4. 통합에 사용된 `active` + `partial` 원본도 `pending/pending-delete.md`에 등록한다
   (stable 파일이 대체하므로)
5. 사용자에게 `/file:delete`를 안내한다

## 출력 형식

```
## Phase 3: 통합 완료

| 항목 | 결과 |
|------|------|
| 원본 세션 | N개 |
| 통합 파일 | sessions/YYYY-MM-DD-stable.md |
| 삭제 대기 등록 | N개 → pending/pending-delete.md |
| stable 파일 크기 | N줄 |

삭제하려면 `/file:delete` 실행
```

## 안전 규칙

- Phase 2 판정 결과를 사용자에게 보여준다. 사용자가 이미 통합을 지시한 경우 재확인 없이 Phase 3으로 진행한다
- 원본 파일을 직접 삭제하지 않는다 — `pending/pending-delete.md`에 등록만
- stable 파일에 세션 원문을 그대로 넣지 않는다 — 요약·압축만
- make/ 변경 이력이 있으면 세션 대신 make/ 참조로 대체한다
