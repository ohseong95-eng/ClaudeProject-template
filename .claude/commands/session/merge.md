# 세션 병합 및 재정리

여러 세션 파일을 하나로 합쳐서 요약본을 생성합니다.

## 인자

- `$ARGUMENTS` — 병합할 세션 번호 (쉼표 또는 공백 구분)
- 예: `/session:merge 1,2,3`
- 인자 없이 실행하면 **전체 세션**을 병합 대상으로 한다

## 실행

1. `sessions/` 폴더의 `.md` 파일을 날짜순(오래된 순)으로 정렬하여 번호를 매긴다 (`.gitkeep` 제외)
2. `$ARGUMENTS`에서 번호를 파싱한다 (없으면 전체)
3. 대상 세션 파일을 모두 읽는다
4. 아래 템플릿에 맞게 **전체 흐름을 시간순으로 재정리**한다
5. `sessions/YYYY-MM-DD-merged.md`로 저장한다
6. 원본 파일은 **삭제하지 않는다** (사용자가 `/session:delete`로 직접 정리)

## 출력 템플릿

```markdown
# 세션 통합 정리: YYYY-MM-DD

**병합 대상**: #1, #2, #3 (3개 세션)
**기간**: YYYY-MM-DD ~ YYYY-MM-DD
**총 Turns**: N

---

## 타임라인 요약

### YYYY-MM-DD (세션 #1)
- 주요 논의: ...
- 결정 사항: ...
- 완료 작업: ...

### YYYY-MM-DD (세션 #2)
- 주요 논의: ...
- 결정 사항: ...
- 완료 작업: ...

---

## 전체 결정 사항
- (모든 세션에서 확정된 결정을 통합 정리)

## 전체 완료 작업
- (모든 세션에서 완료된 작업을 통합 정리)

## 미해결 사항
- (아직 남아있는 항목)

## 수정된 파일 목록
- (전체 세션에서 생성/수정된 파일 통합)
```

## 규칙

- 원본 대화를 그대로 이어붙이지 않는다 — **요약·재정리**하여 읽기 쉽게 만든다
- 중복 내용은 제거한다
- 시간순으로 정렬한다
- 원본 세션 파일은 삭제하지 않는다
