# 파일/디렉토리 삭제

`pending/pending-delete.md`의 삭제 대기 목록을 확인하고 실제 삭제를 수행한다.

**이 커맨드만이 파일/디렉토리를 삭제할 수 있다. 대화 중 직접 rm 명령을 실행하지 않는다.**

## 인자

- `$ARGUMENTS` — 삭제할 항목 번호 (쉼표 또는 공백 구분, 또는 `all`)
- 예: `/file:delete 1,3,5` 또는 `/file:delete all`
- 인자 없이 실행하면 대기 목록만 표시한다

## 실행 절차

### 인자 없이 실행 시 — 목록 조회

1. `pending/pending-delete.md`를 읽는다
2. 대기 항목을 번호와 함께 표시한다
3. "삭제하려면 `/file:delete <번호>` 또는 `/file:delete all`을 실행하세요" 안내

### 번호 지정 실행 시 — 삭제 수행

1. `pending/pending-delete.md`를 읽는다
2. 지정된 번호의 대상 파일/디렉토리 목록을 표시한다
3. 즉시 삭제한다 (사용자가 커맨드를 실행한 것 자체가 삭제 지시)
   - 파일: `rm <경로>`
   - 빈 디렉토리: `rmdir <경로>`
   - 내용이 있는 디렉토리: `rm -r <경로>`
5. 삭제된 항목을 `pending/pending-delete.md`에서 제거한다
6. 결과를 출력한다

## 출력 형식

```
## 삭제 대상

| # | 대상 | 사유 |
|---|------|------|
| 1 | `some/file.md` | 사유 |
| 3 | `some/dir/` | 사유 |

## 삭제 완료

| # | 대상 | 결과 |
|---|------|------|
| 1 | `some/file.md` | 삭제됨 |
| 3 | `some/dir/` | 삭제됨 |
```

## 안전 규칙

- 사용자가 커맨드를 실행한 것 자체가 삭제 지시이므로 재확인하지 않는다
- `pending/pending-delete.md`에 등록되지 않은 파일은 삭제할 수 없다
- 삭제 후 `pending/pending-delete.md`를 갱신하여 완료된 항목을 제거한다
